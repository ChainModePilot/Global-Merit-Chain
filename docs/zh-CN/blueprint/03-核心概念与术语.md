# 第三章 核心概念与术语

## 3.1 实体关系总览

```
自然人（人类原型）
  └── HumanID（全网唯一）
        ├── iFay-1（永久绑定）
        │     ├── FayID
        │     └── MeritPocket → MeriToken（多个批次）
        ├── iFay-2（永久绑定）
        │     ├── FayID
        │     └── MeritPocket → MeriToken（多个批次）
        └── ...

组织/个人
  └── coFay（归属关系，可转移）
        ├── FayID
        └── MeritPocket → MeriToken（多个批次）
```

## 3.2 核心术语

### MeriToken

贡献度的量化单位。代表一个实体的社会公信力和话语权。

- 不可交易、不可转让
- 随时间按指数曲线衰减
- 有不可降至零的底部值
- 可在严格规则下被继承（带折损）

### MeritPocket

MeriToken 的容器，类似钱包。每个 Fay 绑定一个 MeritPocket。

### iFay（个人 Fay）

个人 AI 代理，即"数字盔甲"。与一个自然人永久绑定，不可解绑。

- 本质：人格延伸，不是资产
- iFay 产生的 MeriToken 归属其人类原型
- 一个人可以有多个 iFay

### coFay（组织 Fay）

组织或商业 AI 代理。归属于某个个人或组织。

- 本质：资产，可转移
- coFay 产生的 MeriToken 归属其当前归属者
- 转移时 MeritPocket 随之转移，MeriToken 不折损

### 人类原型

iFay 永久绑定的自然人实体。每个人类原型拥有唯一的 HumanID。

### HumanID / FayID

- HumanID：全网唯一的人类身份标识
- FayID：全网唯一的 Fay 身份标识
- 一个 HumanID 可对应多个 FayID
- HumanID 和 FayID 成对出现

### Merit 批次

每一次贡献获取的记录单元，包含：获取数量、影响期限、衰减参数、获取时间。

### 干系人

与某次贡献相关的利益相关方。负责对贡献进行共识投票，选取时排除与贡献者亲密度过高的人。

### 数字墓园

人类原型过世后，iFay 可被移入的状态。墓园中的 iFay 仍可能有被动互动，但所有行为标注"来自数字墓园"。

## 3.3 iFay 与 coFay 的本质区别

| 维度 | iFay | coFay |
|------|------|-------|
| 本质 | 人格延伸 | 资产 |
| 绑定关系 | 永久绑定，不可解绑 | 归属关系，可转移 |
| 人类原型过世后 | 进入监护或数字墓园 | 作为资产被继承/转让 |
| 转移时 MeriToken | 不可转移 | 随 coFay 一起转移，不折损 |
| 归属者数量 | 只归属一个自然人 | 归属一个个人或组织 |

## 3.4 MeriToken 与 Soulbound Token（SBT）

SBT 是 Vitalik Buterin 于 2022 年提出的概念——不可转让的 Token，绑定到特定身份，用于表示不应被交易的属性（资质、声誉、成就）。

MeriToken 是 SBT 的增强版本：

| 特性 | 标准 SBT | MeriToken |
|------|----------|-----------|
| 不可转让 | ✓ | ✓ |
| 绑定方式 | 绑定钱包地址 | 绑定 iFay → MeritPocket → 人类原型 |
| 时间维度 | 无（永久有效） | 有（指数衰减） |
| 可继承 | 否 | 是（带折损） |
| 量化方式 | 通常为布尔值（有/无） | 连续数值 |
| 底部保障 | 无 | 有（minMerit） |

## 3.5 讨论备注

> 术语体系的设计逻辑：
> - 三层绑定（人类原型 → iFay → MeritPocket）隔离了身份、代理、资产三个层面
> - iFay 作为"人格延伸"不可转移，保证了信誉与人的不可分离
> - coFay 作为"资产"可转移，保证了组织运营的灵活性
> - MeriToken 参考 SBT 但增加了时间衰减和继承性，更适合动态的贡献度量场景
