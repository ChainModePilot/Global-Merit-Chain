# Глава 3: Основные концепции и Терминология

## 3.1 Обзор связей между сущностями

```
Физическое лицо (человеческий архетип)
  └── HumanID (глобально уникальный)
        ├── iFay-1 (постоянно привязан)
        │     ├── FayID
        │     └── MeritPocket → MeriToken (несколько партий)
        ├── iFay-2 (постоянно привязан)
        │     ├── FayID
        │     └── MeritPocket → MeriToken (несколько партий)
        └── ...

Организация / Физическое лицо
  └── coFay (отношение собственности, передаваемый)
        ├── FayID
        └── MeritPocket → MeriToken (несколько партий)
```

## 3.2 Основная терминология

### MeriToken

Количественная единица вклада. Представляет социальную достоверность и право голоса сущности.

- Не подлежит обмену, не передаётся
- Затухает со временем по экспоненциальной кривой
- Имеет минимальное значение, которое не может упасть до нуля
- Может наследоваться по строгим правилам (с ослаблением)

### MeritPocket

Контейнер для MeriToken, аналог кошелька. Каждый Fay привязан к одному MeritPocket.

### iFay (Персональный Fay)

Персональный ИИ-агент — «цифровая броня». Постоянно привязан к физическому лицу; не может быть отвязан.

- Сущность: Расширение личности, а не актив
- MeriToken, сгенерированный iFay, принадлежит его человеческому архетипу
- Человек может иметь несколько iFay

### coFay (Организационный Fay)

Организационный или коммерческий ИИ-агент. Принадлежит физическому лицу или организации.

- Сущность: Актив, передаваемый
- MeriToken, сгенерированный coFay, принадлежит его текущему владельцу
- При передаче MeritPocket передаётся вместе с ним; MeriToken не ослабляется

### Человеческий архетип

Физическое лицо, к которому постоянно привязан iFay. Каждый человеческий архетип обладает уникальным HumanID.

### HumanID / FayID

- HumanID: Глобально уникальный идентификатор личности человека
- FayID: Глобально уникальный идентификатор Fay
- Один HumanID может соответствовать нескольким FayID
- HumanID и FayID появляются парами

### Партия Merit

Единица записи каждого приобретения вклада, содержащая: полученное количество, длительность влияния, параметры затухания и время приобретения.

### Заинтересованные стороны

Стороны, имеющие интерес в данном вкладе. Отвечают за консенсусное голосование по вкладам; отбираются путём исключения лиц с чрезмерно высокой близостью к вкладчику.

### Цифровое кладбище

Состояние, в которое может быть переведён iFay после смерти его человеческого архетипа. iFay на цифровом кладбище может по-прежнему иметь пассивные взаимодействия, но все действия помечаются «с цифрового кладбища».

## 3.3 Принципиальные различия между iFay и coFay

| Измерение | iFay | coFay |
|-----------|------|-------|
| Сущность | Расширение личности | Актив |
| Связь привязки | Постоянно привязан, не может быть отвязан | Отношение собственности, передаваемый |
| После смерти человеческого архетипа | Переходит в опекунство или на цифровое кладбище | Наследуется/передаётся как актив |
| MeriToken при передаче | Не может быть передан | Передаётся вместе с coFay, без ослабления |
| Количество владельцев | Принадлежит только одному физическому лицу | Принадлежит одному физическому лицу или организации |

## 3.4 MeriToken и Soulbound Token (SBT)

SBT — это концепция, предложенная Виталиком Бутериным в 2022 году — непередаваемый Token, привязанный к конкретной личности, используемый для представления атрибутов, которые не должны продаваться (удостоверения, репутация, достижения).

MeriToken — это улучшенная версия SBT:

| Характеристика | Стандартный SBT | MeriToken |
|----------------|-----------------|-----------|
| Непередаваемый | ✓ | ✓ |
| Способ привязки | Привязан к адресу кошелька | Привязан к iFay → MeritPocket → человеческий архетип |
| Временное измерение | Нет (действует бессрочно) | Да (экспоненциальное затухание) |
| Наследуемый | Нет | Да (с ослаблением) |
| Метод квантификации | Обычно булевый (есть/нет) | Непрерывное числовое значение |
| Гарантия минимума | Нет | Да (minMerit) |

## 3.5 Заметки к обсуждению

> Логика проектирования терминологической системы:
> - Трёхуровневая привязка (человеческий архетип → iFay → MeritPocket) изолирует уровни идентичности, агента и активов
> - iFay как «расширение личности» непередаваем, что гарантирует неотделимость репутации от человека
> - coFay как «актив» передаваем, что обеспечивает операционную гибкость организаций
> - MeriToken ссылается на SBT, но добавляет временное затухание и наследуемость, что делает его более подходящим для сценариев динамического измерения вклада
