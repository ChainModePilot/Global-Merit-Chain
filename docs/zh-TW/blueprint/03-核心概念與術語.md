# 第三章 核心概念與術語

## 3.1 實體關係總覽

```
自然人（人類原型）
  └── HumanID（全網唯一）
        ├── iFay-1（永久綁定）
        │     ├── FayID
        │     └── MeritPocket → MeriToken（多個批次）
        ├── iFay-2（永久綁定）
        │     ├── FayID
        │     └── MeritPocket → MeriToken（多個批次）
        └── ...

組織/個人
  └── coFay（歸屬關係，可轉移）
        ├── FayID
        └── MeritPocket → MeriToken（多個批次）
```

## 3.2 核心術語

### MeriToken

貢獻度的量化單位。代表一個實體的社會公信力和話語權。

- 不可交易、不可轉讓
- 隨時間按指數曲線衰減
- 有不可降至零的底部值
- 可在嚴格規則下被繼承（帶折損）

### MeritPocket

MeriToken 的容器，類似錢包。每個 Fay 綁定一個 MeritPocket。

### iFay（個人 Fay）

個人 AI 代理，即「數位盔甲」。與一個自然人永久綁定，不可解綁。

- 本質：人格延伸，不是資產
- iFay 產生的 MeriToken 歸屬其人類原型
- 一個人可以有多個 iFay

### coFay（組織 Fay）

組織或商業 AI 代理。歸屬於某個個人或組織。

- 本質：資產，可轉移
- coFay 產生的 MeriToken 歸屬其當前歸屬者
- 轉移時 MeritPocket 隨之轉移，MeriToken 不折損

### 人類原型

iFay 永久綁定的自然人實體。每個人類原型擁有唯一的 HumanID。

### HumanID / FayID

- HumanID：全網唯一的人類身份標識
- FayID：全網唯一的 Fay 身份標識
- 一個 HumanID 可對應多個 FayID
- HumanID 和 FayID 成對出現

### Merit 批次

每一次貢獻取得的記錄單元，包含：取得數量、影響期限、衰減參數、取得時間。

### 幹係人

與某次貢獻相關的利害關係方。負責對貢獻進行共識投票，選取時排除與貢獻者親密度過高的人。

### 數位墓園

人類原型過世後，iFay 可被移入的狀態。墓園中的 iFay 仍可能有被動互動，但所有行為標註「來自數位墓園」。

## 3.3 iFay 與 coFay 的本質區別

| 維度 | iFay | coFay |
|------|------|-------|
| 本質 | 人格延伸 | 資產 |
| 綁定關係 | 永久綁定，不可解綁 | 歸屬關係，可轉移 |
| 人類原型過世後 | 進入監護或數位墓園 | 作為資產被繼承/轉讓 |
| 轉移時 MeriToken | 不可轉移 | 隨 coFay 一起轉移，不折損 |
| 歸屬者數量 | 只歸屬一個自然人 | 歸屬一個個人或組織 |

## 3.4 MeriToken 與 Soulbound Token（SBT）

SBT 是 Vitalik Buterin 於 2022 年提出的概念——不可轉讓的 Token，綁定到特定身份，用於表示不應被交易的屬性（資質、聲譽、成就）。

MeriToken 是 SBT 的增強版本：

| 特性 | 標準 SBT | MeriToken |
|------|----------|-----------|
| 不可轉讓 | ✓ | ✓ |
| 綁定方式 | 綁定錢包地址 | 綁定 iFay → MeritPocket → 人類原型 |
| 時間維度 | 無（永久有效） | 有（指數衰減） |
| 可繼承 | 否 | 是（帶折損） |
| 量化方式 | 通常為布林值（有/無） | 連續數值 |
| 底部保障 | 無 | 有（minMerit） |

## 3.5 討論備註

> 術語體系的設計邏輯：
> - 三層綁定（人類原型 → iFay → MeritPocket）隔離了身份、代理、資產三個層面
> - iFay 作為「人格延伸」不可轉移，保證了信譽與人的不可分離
> - coFay 作為「資產」可轉移，保證了組織營運的靈活性
> - MeriToken 參考 SBT 但增加了時間衰減和繼承性，更適合動態的貢獻度量場景
