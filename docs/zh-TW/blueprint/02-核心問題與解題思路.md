# 第二章 核心問題與解題思路

## 2.1 問題全景

後貨幣時代面臨的核心問題可以歸納為一個根本矛盾：

**當生存不再是勞動的動力，社會如何激勵貢獻、記錄貢獻、並基於貢獻進行治理？**

這個根本矛盾分解為四個子問題：

### 問題一：激勵

沒有貨幣回報，人們為什麼還要付出？

答案：社會認同。人類天然渴望被認可、被尊重、擁有影響力。GMC 將貢獻轉化為可見的、可量化的社會認同（MeriToken），並賦予其實際的話語權。

### 問題二：度量

貢獻如何被公正地記錄和衡量？

答案：去中心化的共識記錄。藉助區塊鏈和加密技術，建立不可偽造、不可竄改、不可否認的貢獻記錄網路。沒有任何單一機構可以扭曲記錄。

### 問題三：治理

話語權應該基於什麼分配？

答案：基於貢獻度的動態分配。話語權來自 MeriToken 在社群中的佔比，而 MeriToken 會衰減——這意味著話語權必須透過持續貢獻來維持，不存在永久的特權。

### 問題四：人機協同

當 Fay 承擔了大部分工作，如何公平衡量人類和 Fay 的貢獻？

答案：統一的度量體系。人類和 Fay 在同一個系統中被記錄和衡量，Fay 的貢獻歸屬於其人類原型或所屬組織。

## 2.2 解題思路

GMC 的解題思路可以概括為：

```
貢獻 → 記錄 → 度量 → 話語權
```

1. **貢獻發生**：人類或 Fay 做出了對社會有價值的行為
2. **貢獻被記錄**：透過共識機制，貢獻被不可竄改地記錄在鏈上
3. **貢獻被度量**：貢獻轉化為 MeriToken，有衰減、有底部值
4. **話語權生效**：MeriToken 佔比決定在社群治理中的影響力

## 2.3 六大設計原則

| 原則 | 含義 | 解決的問題 |
|------|------|-----------|
| 不可偽造 | 貢獻記錄必須經過共識驗證 | 防止虛假貢獻 |
| 不可竄改 | 歷史記錄永久保存，無法修改 | 保證記錄公信力 |
| 不可否認 | 貢獻者無法否認自己的行為 | 責任可追溯 |
| 去中心化 | 不依賴單一機構 | 防止權力壟斷 |
| 動態衡量 | 貢獻度隨時間衰減 | 激勵持續參與 |
| 全民參與 | 面向所有人類和 Fay | 公平性 |

## 2.4 與貨幣體系的本質區別

| 維度 | 貨幣 | MeriToken |
|------|------|-----------|
| 本質 | 交換媒介 | 貢獻度量 |
| 驅動力 | 生存需求 | 社會認同與自我實現 |
| 可交易性 | 可自由交易 | 不可交易、不可轉讓 |
| 時代背景 | 物資稀缺時代 | 物資充裕時代 |
| 分配邏輯 | 按支付能力分配資源 | 按貢獻度分配話語權 |
| 累積效應 | 可無限累積（貧富分化） | 自然衰減（防止固化） |

## 2.5 討論備註

> 本章建立的邏輯框架：
> - 從「後貨幣時代的根本矛盾」出發，推導出 GMC 的必要性
> - 四個子問題（激勵、度量、治理、人機協同）構成了 GMC 的設計目標
> - 「貢獻 → 記錄 → 度量 → 話語權」是整個系統的核心流轉邏輯
> - MeriToken 與貨幣的本質區別在於：貨幣解決分配問題，MeriToken 解決認同和治理問題
