# 제3장: 핵심 개념과 용어 정의

## 3.1 엔티티 관계 개요

```
자연인 (인간 원형)
  └── HumanID (전역 고유)
        ├── iFay-1 (영구 바인딩)
        │     ├── FayID
        │     └── MeritPocket → MeriToken (다수의 배치)
        ├── iFay-2 (영구 바인딩)
        │     ├── FayID
        │     └── MeritPocket → MeriToken (다수의 배치)
        └── ...

조직 / 개인
  └── coFay (소유 관계, 양도 가능)
        ├── FayID
        └── MeritPocket → MeriToken (다수의 배치)
```

## 3.2 핵심 용어

### MeriToken

기여의 정량적 단위입니다. 엔티티의 사회적 신뢰도와 발언권을 나타냅니다.

- 거래 불가, 양도 불가
- 지수 곡선에 따라 시간이 지남에 따라 감쇠
- 0으로 떨어질 수 없는 하한값 보유
- 엄격한 규칙 하에 상속 가능 (감쇠 적용)

### MeritPocket

MeriToken의 컨테이너로, 지갑에 비유할 수 있습니다. 각 Fay는 하나의 MeritPocket에 바인딩됩니다.

### iFay (개인 Fay)

개인 AI 에이전트 — "디지털 갑옷"입니다. 자연인에게 영구적으로 바인딩되며; 해제할 수 없습니다.

- 본질: 인격의 확장이며, 자산이 아닙니다
- iFay가 생성한 MeriToken은 그 인간 원형에 귀속됩니다
- 한 사람이 여러 iFay를 가질 수 있습니다

### coFay (조직 Fay)

조직 또는 상업용 AI 에이전트입니다. 개인 또는 조직에 소속됩니다.

- 본질: 자산이며, 양도 가능합니다
- coFay가 생성한 MeriToken은 현재 소유자에게 귀속됩니다
- 양도 시 MeritPocket이 함께 이전되며; MeriToken은 감쇠되지 않습니다

### 인간 원형

iFay가 영구적으로 바인딩된 자연인입니다. 각 인간 원형은 고유한 HumanID를 보유합니다.

### HumanID / FayID

- HumanID: 전역 고유 인간 신원 식별자
- FayID: 전역 고유 Fay 신원 식별자
- 하나의 HumanID는 여러 FayID에 대응할 수 있습니다
- HumanID와 FayID는 쌍으로 나타납니다

### 메리트 배치

각 기여 획득의 기록 단위로, 다음을 포함합니다: 획득 수량, 영향 지속 기간, 감쇠 매개변수, 획득 시간.

### 이해관계자

특정 기여에 이해관계가 있는 당사자들입니다. 기여에 대한 합의 투표를 담당하며; 기여자와 친밀도가 과도하게 높은 개인을 제외하여 선정됩니다.

### 디지털 묘원

인간 원형이 사망한 후 iFay가 배치될 수 있는 상태입니다. 디지털 묘원에 있는 iFay는 여전히 수동적 상호작용을 할 수 있지만, 모든 행동에는 "디지털 묘원에서"라는 라벨이 표시됩니다.

## 3.3 iFay와 coFay의 본질적 차이

| 차원 | iFay | coFay |
|------|------|-------|
| 본질 | 인격의 확장 | 자산 |
| 바인딩 관계 | 영구 바인딩, 해제 불가 | 소유 관계, 양도 가능 |
| 인간 원형 사망 후 | 후견 또는 디지털 묘원 진입 | 자산으로서 상속/양도 |
| 양도 시 MeriToken | 양도 불가 | coFay와 함께 이전, 감쇠 없음 |
| 소유자 수 | 한 자연인에게만 귀속 | 한 개인 또는 조직에 귀속 |

## 3.4 MeriToken과 Soulbound Token (SBT)

SBT는 2022년 비탈릭 부테린이 제안한 개념으로 — 특정 신원에 바인딩된 양도 불가능한 Token이며, 거래되어서는 안 되는 속성(자격증, 평판, 업적)을 나타내는 데 사용됩니다.

MeriToken은 SBT의 강화 버전입니다:

| 특성 | 표준 SBT | MeriToken |
|------|----------|-----------|
| 양도 불가 | ✓ | ✓ |
| 바인딩 방식 | 지갑 주소에 바인딩 | iFay → MeritPocket → 인간 원형에 바인딩 |
| 시간 차원 | 없음 (영구 유효) | 있음 (지수 감쇠) |
| 상속 가능 | 아니오 | 예 (감쇠 적용) |
| 정량화 방식 | 일반적으로 불리언 (있음/없음) | 연속적 수치 |
| 하한 보장 | 없음 | 있음 (minMerit) |

## 3.5 논의 메모

> 용어 체계의 설계 논리:
> - 3계층 바인딩(인간 원형 → iFay → MeritPocket)은 신원, 에이전트, 자산 계층을 분리합니다
> - iFay는 "인격의 확장"으로서 양도 불가능하며, 평판이 개인과 분리될 수 없음을 보장합니다
> - coFay는 "자산"으로서 양도 가능하며, 조직의 운영 유연성을 보장합니다
> - MeriToken은 SBT를 참조하되 시간 감쇠와 상속 가능성을 추가하여, 동적 기여 측정 시나리오에 더 적합합니다
